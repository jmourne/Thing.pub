# Thing.pub
**Secure Manuscript Distribution | Line-Level Feedback | Reader Intelligence**

## The Mission
This platform is a proprietary environment built for authors and their manuscripts. It bridges the gap between extreme IP protection and deep reader analytics, allowing authors to observe how readers engage with their work while gathering granular, line-level critiques.

## Core Feature Suite

### 1. Granular Feedback Engine
- **Line-Level Commenting:** Move beyond general impressions. Readers can highlight specific sentences or phrases for feedback.
- **Threaded Author Dialogue:** Engage directly with your readers within the manuscript to clarify notes and iterate on the fly.
- **Private Access:** Each reader has their own access, so all feedback can remain unbiased and without influence of other reader's comments. 
- **Compiled Sentiment:** A centralized dashboard for the author to review all reader feedback, categorized by book, chapter, intensity, and context.

### 2. Reader Analytics & Intelligence
- **Engagement Heatmaps:** Track exactly where readers slow down, skip, or re-read sections.
- **Dwell Time Tracking:** Detailed metrics on how long a reader spends on each chapter or scene.
- **Completion Funnels:** Monitor the drop-off rate to ensure the manuscript maintains momentum through the final act.
- **Review Prompts:** To increase the rate of review completion (ARC readers), readers are prompted with a review request as soon as they complete the manuscript, when the story is still fresh. The platform saves their review and an automated reminder (configured by authors) is sent to remind them to post their review on release day or as soon as leaving reviews across multiple sites is available. 

### 3. Forensic Security
- **Device Locking:** Hardware fingerprinting restricts each access code to a maximum of 2 unique devices.
- **Dynamic Watermarking:** A non-removable, tiled overlay of the reader's unique access code ensures total accountability. If a leak does occur, you know exactly who is responsible. 
- **Anti-Piracy Hardening:** Globally disabled text selection, right-click, and print-shielding.
- **No Downloads:** Because most security breaches happen on local devices, there is no download option for readers. All reading and feedback is contained within the secure app.

## Contributor Portal

I value contributors who understand the intersection of high-end UX and data security. In order to maintain highest possible standards, contribution to this project is by invitation only. if you're interested in being a community contributor, please submit the request form here.

### How to Contribute
1. **Feature Requests:** Open an Issue tagged `feature-request` for new analytics or UI refinements.
2. **Security Audits:** If you identify a potential leak in RLS or the watermarking layer, report it via a private message immediately.
3. **App Feedback:** I utilize a dedicated internal feedback loop. If you are an authorized contributor, please submit your UX/UI feedback via the **`INTERNAL_FEEDBACK.md`** file or the `/admin/feedback` route.

### Development Standards
- **Privacy First:** No telemetry. No third-party trackers. No data leaves our Supabase/Vercel perimeter.
- **Design & UX:** All typography must remain Serif-heavy (Playfair Display/Merriweather) to honor the long-form reading experience. Following the style and brand guide is mandatory for front-end contributions.
- **Clean Git History:** Meaningful commit messages and atomic pull requests are mandatory.
